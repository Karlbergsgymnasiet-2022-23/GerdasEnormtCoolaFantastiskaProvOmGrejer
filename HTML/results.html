Vad betyder GDPR?
Ditt svar:

Rätt svar: General Data Protection Regulation

Vad bestämmer GDPR om?
Ditt svar:

Rätt svar: GDPR-lagen är en europeisk förordning som innefattar hur data så som personuppgifter ska hanteras inom Europeiska unionen. Den omfattar hur personers integritet ska hanteras. Lagen har obligatoriska regler för hur organisationer och företag får använda personuppgifter på ett lämpligt sätt. Personuppgifter är information som kan indirekt eller direkt kopplas till en person och kan exempelvis bestå av namn, telefonnummer och adress. Denna tillämpas både på internet samt utanför det.

När ska du använda metoden POST för att skicka data?
Ditt svar:

Rätt svar: POST används för att skicka information till en server för att skapa eller uppdatera en resurs.

När ska du använda metoden GET för att skicka data?
Ditt svar:

Rätt svar: GET används för att begära data från en specificerad resurs.

Ska du som utvecklare lita på användare? Resonera utifrån ett säkerhetsperspektiv!

Rätt svar: Nej, eftersom vi ej vet användarens intentioner. Användaren kan vilja orsaka fel i mjukvaran eller så vet den ej hur hen ska använda mjukvaran. Då är det bäst att använda metoder för att eliminera potentiella fel. Exempelvis kan användaren vilja hacka programmet eller orsaka andra problem eller rentav okunnig. Ska du som användare lita på utvecklare? Resonera utifrån ett säkerhetsperspektiv!
Ditt svar:

Rätt svar: Man kan lita på utvecklare till viss del, men man ska alltid vara källkritisk mot de sidor man möter på internet. Detta eftersom utvecklare kan vara bedragare eller vilja ta dina uppgifter för sitt eget bästa.

Hur kan du som utvecklare skydda användares lösenord?
Ditt svar:

Rätt svar:

  • Kryptering
  • Hashning
  • Skapa en säker databas
  • Kräva en viss minimilängd på lösenord

Redogör för dessa sätt.
Ditt svar:

Rätt svar: Kryptering och hashning gör att data ej kan avläsas så lätt. Minimilängd på lösenord förhindrar att användaren väljer ett för enkelt lösenord som är lätt för hackare att gissa. En säker databas gör att datan skyddas när den lagts till och förhindrar att den råkar läcka ut.

Hur kan man skapa säkra lösenord?
Ditt svar:

Rätt svar: För att skapa säkra lösenord kan man använda lösenordshanterare, eller skriva om ett slumpmässigt ord med versaler, gemener, skiljetecken och siffror.

Vad är skillnaden mellan kryptering och hashning?
Ditt svar:

Rätt svar: Kryptering är en funktion som gör en text oläslig som kan bli läsbar efter att den dekrypterats. Nycklar tillämpas för hur någonting ska krypteras. Hashning är en funktion som gör en text oläslig och som ej kan dekrypteras. Den producerar en uppsättning av tecken utefter en indata.

Beskriv varför det är en dum ide att lagra saker i klartext.
Ditt svar:

Rätt svar: Därför att det blir lättare för hackare att ta reda på känsliga uppgifter, vilket man inte vill i de flesta sammanhang. Att lagra något krypterat eller med hash skyddar användarens uppgifter och motverkar att data läcker ut vid hack-attacker.

Beskriv vad som händer vid XSS.
Ditt svar:

Rätt svar: Det som händer vid XSS är att man kan anropa egna skrift genom utvecklarens utskrift. Exempelvis kan JavaScript läggas till i en kommentar och orsaka konsekvenser för webbsidor.
